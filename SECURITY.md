# Security Policy

## Supported use

Iron Council is an active public repository. Security fixes will be handled in the main branch and released through normal repository updates.

## Reporting a vulnerability

If you find a security issue:

1. Do not post exploit details, proof-of-concept code, or reproduction steps in a public issue.
2. Use GitHub's private vulnerability reporting or repository security advisory flow if it is available for this repository.
3. If no private reporting path is available, open a minimal public issue that states a potential security problem exists and asks maintainers to coordinate next steps without disclosing sensitive details.

For non-sensitive hardening suggestions, regular GitHub issues are fine.

## What to include

When you report a vulnerability, include:

- Affected component or path
- Impact and attack conditions
- Reproduction steps or proof details, but only in a private channel when one exists
- Suggested mitigation if you have one

## Response expectations

Maintainers will triage reports as time allows, confirm severity, and prefer a fix-first disclosure flow.

## Scope notes

Please keep reports focused on the Iron Council code and repository configuration. Third-party service vulnerabilities should usually be reported to the affected upstream project as well.
